# Glossary

Canonical definitions for terms and acronyms used across Canari's documentation and codebase.

## Cryptographic

| Term | Definition |
|---|---|
| **MLS** | Messaging Layer Security (RFC 9420). The end-to-end encryption protocol used for all direct messages and group chats. |
| **CEK** | Content Encryption Key. A random AES-256-GCM key generated by the client to encrypt a media file before upload. Travels inside the MLS ciphertext; the server never sees it. |
| **E2EE** | End-to-End Encryption. Plaintext is only readable by the sender and intended recipients. |
| **HKDF** | HMAC-based Key Derivation Function. Used to derive per-channel encryption keys from a workspace master secret. |
| **HS256** | HMAC with SHA-256. The JWT signing algorithm used for access tokens. |
| **PBKDF2** | Password-Based Key Derivation Function 2. Used for PIN verifier hashing (server-side) and MLS state encryption key derivation (client-side). |
| **PKCE** | Proof Key for Code Exchange. OAuth 2.0 extension used in the OIDC login flow to prevent authorization code interception. |
| **Forward secrecy** | Compromise of current keys does not reveal past messages. MLS achieves this per epoch via key ratcheting. |
| **Post-compromise security** | A compromised device can be removed from a group and re-added with fresh keys, restoring security. |

## MLS concepts

| Term | Definition |
|---|---|
| **KeyPackage** | A public key bundle published by each device. Contains the device's identity credential and encryption key. Used by other members to add the device to a group. |
| **OTKP** | One-Time Key Package. A single-use prekey consumed when a device is added to a group. Pool of 50 (web) or 200 (Tauri). |
| **Welcome** | A message containing the group secrets encrypted to a new member's KeyPackage. Processing a Welcome joins the device to the group. |
| **Commit** | An MLS protocol message that advances the group epoch. Carries structural changes: add/remove members, key updates. |
| **Epoch** | A monotonic counter that increments on every commit. Each epoch has a distinct key, providing forward secrecy. |
| **Ratchet tree** | The public group state (member public keys) needed by a new member to process messages. Delivered alongside the Welcome. |
| **GroupInfo** | A snapshot of the group state used for external joins (self-service recovery). Stored server-side, refreshed after each commit. |
| **External join** | Self-service recovery: a device fetches GroupInfo and builds an external commit to rejoin a group without needing a peer to send a Welcome. |
| **Commit-log** | Server-side ordered log of commit bytes per group. Used for rung-1 epoch gap recovery (replay missed commits). |

## Infrastructure

| Term | Definition |
|---|---|
| **OIDC** | OpenID Connect. Authentication layer on top of OAuth 2.0. Canari uses Authentik as the OIDC provider. |
| **JWT** | JSON Web Token. HS256-signed access tokens (15 min TTL), stored in memory only. |
| **SFU** | Selective Forwarding Unit. The WebRTC media relay (`call-service`) that forwards encrypted RTP between call participants without decrypting. |
| **TURN** | Traversal Using Relays around NAT. A relay server that proxies media when direct peer-to-peer connections fail. Production uses Cloudflare TURN. |
| **STUN** | Session Traversal Utilities for NAT. Used to discover public IP/port for WebRTC. |
| **PLI** | Picture Loss Indication. An RTCP feedback message requesting a video keyframe from the sender. |
| **FIR** | Full Intra Request. Similar to PLI, requests a complete refresh of the video stream. |
| **NSE** | Notification Service Extension. An iOS app extension that runs briefly when a push arrives (app killed) to decrypt and render the notification. |
| **FCM** | Firebase Cloud Messaging. Google's push notification service. Canari uses FCM for both Android and iOS (FCM relays to APNs). |
| **APNs** | Apple Push Notification service. iOS push delivery. Canari uses FCM→APNs for message pushes and direct APNs for VoIP (CallKit). |

## Services

| Term | Definition |
|---|---|
| **chat-gateway** | Rust/Axum WebSocket gateway. Routes MLS frames to connected devices via Redis pub/sub. |
| **call-service** | Rust/Axum WebRTC SFU. Forwards encrypted RTP between call participants. |
| **chat-delivery-service** | NestJS MLS API. Device registration, group management, message persistence, push notifications. |
| **core-service** | NestJS auth hub. OIDC callback, JWT issuance, user profiles, Stripe payments. |
| **media-service** | NestJS encrypted blob store. Accepts and serves opaque ciphertext via MinIO. |
| **social-service** | NestJS community features. Posts, channels, associations, forms, cotisations. |
| **MinIO** | S3-compatible object storage used by media-service for encrypted blobs. |
| **Authentik** | Open-source Identity Provider. Deployed as the `miconnect` Docker Compose stack. |

## Concepts

| Term | Definition |
|---|---|
| **Cotisation** | Membership dues. Modelled as a time-bounded `user_tags` row granted by an association, not as a boolean. |
| **Cotisant** | A user who holds an active (non-expired) cotisation tag for an association. |
| **Workspace** | A community container holding multiple channels. Each workspace has a master secret from which per-channel keys are derived. |
| **Channel** | An encrypted group chat inside a workspace. Uses HKDF-derived symmetric keys distributed via MLS. |
| **Dismissed group** | A group the user has explicitly hidden. Stored server-side, synced across devices. |
| **DeviceGroupMembership** | Per-device state machine tracking whether a device is `pending`, `active`, or `removed` in a group. |
| **Add-lock** | A distributed Redis lock preventing concurrent member additions to the same group. |
| **Paraglide** | SvelteKit i18n library. Source messages in `frontend/messages/fr.json`, translated to `en.json`. |
| **Snapdom** | HTML-to-canvas rasterisation library used for PDF exports (calendar, trombinoscope, carte-vie-asso). |
| **Run commit transaction** | The single staged-commit primitive (`runCommitTransaction`) behind every MLS structural change. Stage → validate epoch server-side → merge on accept / rollback on reject. |
